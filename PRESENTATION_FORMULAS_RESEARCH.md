# Presentation Formulas Research for Deck Director

## Core conclusion

The best presentations do **not** come from one magic proportion.
They come from combining four systems:

1. **Composition formulas** — where elements sit on the slide
2. **Hierarchy formulas** — how headline, support, and evidence are weighted
3. **Cognitive formulas** — how much the audience can process per slide
4. **Narrative formulas** — how slides sequence for persuasion and memory

Deck Director should treat these as a **presentation operating system** for AI generation.

---

## 1. Composition formulas

### A. Rule of Thirds
**Use:** Fast, reliable composition guide for hero slides, quote slides, divider slides, image-led narrative slides.

**Why it matters:** Key elements placed on third-lines or intersections usually feel more dynamic than dead-center layouts.

**For Deck Director:**
- place hero subject or key focal area on a third intersection
- place headline block on opposite third region for tension/balance
- avoid perfect center unless intentional and ceremonial

**Best for:**
- cover
- divider
- quote
- narrative with strong image

---

### B. Golden Ratio / Phi-inspired split
**Use:** Premium asymmetric layout balance.

**Important note:** In presentations, the golden ratio should be a **guideline**, not dogma.
It works best as a split heuristic such as:
- 62/38
- 38/62

**For Deck Director:**
- use on text-image slides
- use for premium strategic slides where one side carries weight and the other supports
- use more sparingly than rule of thirds

**Best for:**
- text-image narrative slides
- evidence slides
- opportunity slides
- strategic framing slides

---

### C. Modular grid / editorial grid
**Use:** Structural consistency across the full deck.

**Why it matters:** Great decks feel coherent because content lands on repeatable rails.

**For Deck Director:**
- base all slides on a 12-column grid
- use standard internal zones:
  - headline band
  - support band
  - evidence/media field
  - footer/meta band
- AI should compose inside a grid even when output looks organic

**Best for:** all slide types.

---

### D. Fibonacci / modular scaling
**Use:** Size relationships, not literal spiral worship.

**Practical application:**
Use a modular scale for type, spacing, and block sizing.
Examples:
- 8, 13, 21, 34, 55, 89
- or simpler production scale: 8, 16, 24, 32, 48, 64, 96
- typography multiplier: 1.25 or 1.333

**For Deck Director:**
- keep spacing on fixed steps
- use clear jumps in hierarchy
- avoid random sizes

**Best for:**
- typography
- padding
- margins
- card heights
- rhythm across deck

## 2. Hierarchy formulas

### A. Assertion–Evidence structure
This is huge for presentations.

Instead of:
- vague title
- bullet dump

Use:
- **assertion headline** = the point of the slide
- **evidence visual** = chart, image, framework, comparison, proof
- **minimal support text** = only what clarifies

**Formula:**
`One claim + one evidence field + 0–2 clarifying annotations`

**For Deck Director:**
Every slide should answer:
- what is the point?
- what proves/supports it?
- what can be removed?

**This is probably more important than golden ratio.**

---

### B. 1-slide = 1 idea rule
A slide may contain multiple elements, but it should only communicate **one dominant idea**.

**For Deck Director:**
- if row content contains 3 competing ideas, split or compress
- AI should identify dominant claim before layout generation

---

### C. Signal-to-noise ratio
Borrowed from information design.

**Rule:** maximize meaningful content, minimize decorative or repetitive clutter.

**For Deck Director:**
Avoid:
- repeated labels
- redundant copy
- decorative chrome
- unnecessary icons
- excessive dividers

---

## 3. Cognitive formulas

These are critical because slides are not posters or webpages — they are processed live, in time.

### A. Hick’s Law
The more choices/options shown, the longer it takes to decide/process.

**For presentations:**
- too many bullets, too many cards, too many competing focal points slows comprehension

**Deck Director rule:**
- default max visible choices/items per cluster:
  - comparison: 2–3
  - card framework: 3–5
  - bullet list: 3–5
  - chart callouts: 1–3 primary callouts

Use progressive disclosure across multiple slides instead of one mega-slide.

---

### B. Von Restorff Effect
The item that differs from the rest is more likely to be remembered.

**For presentations:**
Every strong slide should have one clearly isolated memorable thing:
- one bold phrase
- one highlighted number
- one accent color moment
- one unusual image
- one oversized object

**Deck Director rule:**
- allow only **one primary isolate** per slide
- do not highlight everything
- if everything is loud, nothing is memorable

---

### C. Cognitive Load / Mayer principles
This is probably the deepest well for slide quality.

Most relevant multimedia learning principles for presentations:

#### 1. Coherence
Remove irrelevant material.
- kill decorative filler
- cut side facts
- reduce ornamental noise

#### 2. Signaling
Use cues that show structure.
- arrows
- bold contrast
- highlighted stat
- sequencing markers
- section dividers

#### 3. Redundancy
Don’t say the exact same thing in headline, body, and presenter speech.
- slides should support speech, not transcript it

#### 4. Spatial Contiguity
Related text and visuals should be near each other.
- labels should sit close to chart/image regions
- avoid legends that force eye ping-pong

#### 5. Segmenting
Break complex material into chunks.
- one complex process = multiple slides if needed

#### 6. Modality awareness
If presenter is speaking, slide can be lighter.
- visual-heavy slides work well when speech carries detail

**Deck Director implication:**
This should become a formal generation validator.

## 4. Perception / grouping formulas

### A. Gestalt principles
The audience perceives groups before details.

Most useful for presentations:

#### Proximity
Things close together feel related.
Use spacing to group, not just borders.

#### Similarity
Matching style implies shared category.
Use repeated card style, icon treatment, caption style.

#### Continuity
Eyes like smooth directional flow.
Build left→right or top→bottom reading path.

#### Figure/Ground
Important thing must separate from background.
Need strong contrast and clarity.

#### Closure
People can infer incomplete forms.
Useful for elegant minimal framing.

**Deck Director rule:**
Slides should have obvious grouping before the audience starts reading.

## 5. Presentation-specific layout formulas by slide type

### Cover slide
**Formula:**
- 1 dominant visual world
- 1 assertion headline
- minimal support line
- one focal anchor on third or phi split

### Divider slide
**Formula:**
- 1 section title
- massive negative space
- 1 atmosphere image or texture field
- no clutter

### Narrative text-image slide
**Formula:**
- 62/38 or 60/40 split
- one headline
- short body or 3–4 bullets max
- image carries emotion/context

### Framework slide
**Formula:**
- one assertion headline
- 3 or 4 cards max
- equal rhythm and alignment
- one key card emphasized if needed

### Evidence slide
**Formula:**
- one claim headline
- one dominant proof object (chart, stat, comparison)
- 1–3 callouts max
- source/meta small and quiet

### Quote / insight slide
**Formula:**
- one dominant sentence
- one highlighted phrase
- minimal attribution
- big type, strong whitespace

### Summary slide
**Formula:**
- 3 key takeaways max
- one closing line / action / implication
- do not re-dump entire deck

## 6. Rhythm formulas across the deck

Great decks do not keep the same energy every slide.

### Pacing rule
Use variation:
- dense strategic slide
- clean release slide
- emotional image-led slide
- proof slide
- framework slide

**Deck formula:**
Avoid more than 2–3 structurally similar slides in a row unless intentional.

### Contrast cadence
Memory improves when rhythm changes deliberately.
Use alternation between:
- text-led vs image-led
- dark vs light moments
- close-up human vs abstract system
- broad ambition vs concrete proof

---

## 7. What not to over-believe

### Golden Ratio is not a cheat code
It helps with premium asymmetry, but it does not rescue weak content.

### Fibonacci spiral is often over-romanticized
Useful for size relationships and rhythm, not as a literal composition overlay on every slide.

### Web UI laws do not map perfectly
Slides are linear, presenter-assisted, and time-based.
That makes:
- cognitive load
- assertion-evidence
- memory isolation
- sequencing
more important than raw UI mechanics.

## 8. Recommended formula stack for Deck Director

### Tier 1 — Must-have
1. **Assertion–Evidence**
2. **1-slide = 1 idea**
3. **Coherence / signaling / contiguity / segmenting**
4. **Grid system**
5. **One focal isolate per slide (Von Restorff)**

### Tier 2 — Strong visual quality multipliers
6. **Rule of Thirds**
7. **62/38 premium split**
8. **Modular type + spacing scale**
9. **Gestalt grouping principles**

### Tier 3 — Deck-level intelligence
10. **Pacing / rhythm alternation**
11. **Density control across sequence**
12. **Variation with continuity**

---

## 9. Proposed system rules for AI generation

### Composition rules
- default to 12-column grid
- prefer thirds or 62/38 split over centered symmetry
- keep one dominant focal region per slide

### Hierarchy rules
- headline must state the point, not label the topic
- support copy must be subordinate
- proof object must visually outrank filler

### Density rules
- bullets: 3–5 max
- cards: 3–5 max
- key callouts: 1–3 max
- one primary message only

### Memory rules
- one isolated memorable element per slide
- one accent color moment max
- one oversized emphasis zone max

### Continuity rules
- inherit deck DNA from reference slides
- vary pacing, not identity
- carry recurring alignment, margins, and type logic through sequence

---

## 10. Best implementation direction for Deck Director

Deck Director should not merely ask Nano Banana 2 to “make a pretty slide.”
It should generate with an explicit reasoning scaffold:

1. determine slide archetype
2. determine single slide assertion
3. determine evidence object
4. determine composition formula
5. determine focal isolate
6. determine density limit
7. determine continuity with previous 2 slides
8. generate image

That turns the system from prompt art into **presentation intelligence**.
